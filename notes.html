Lesson 19: Understanding Dynamic Websites and HTML5 Applications

What You'll Learn in this Lesson

  • How to conceptualize different types of dynamic content
  • How to include JavaScript in your HTML
  • How to display randomized text with JavaScript
  • How to change images by using JavaScript and user events
  • How to begin thinking ahead to putting all the pieces together to create HTML5 applications

Web development involves server and client-side scripting, both are forms of computer programming. Server-side scripting is a process where scripts run on a web server, sending results to the browser. JavaScript code can be stored in external files or directly in HTML files, with the browser recognizing its existence through the tag pair. <script> tags are placed between <head> tags to provide JavaScript functions or other information for the rest of the page to use. JavaScript allows you to display unique content on each page load, such as interesting text or images. JavaScript client-side interactivity involves manipulating the Document Object Model (DOM), an invisible structure for documents, not an HTML structure or semantic formatting.

The document is the overall container object, and any container created within it with an assigned ID can be referenced by that ID. Unobtrusive JavaScript is a method for writing JavaScript scripts, not necessarily for interactive web pages, but best practices suggest considering it. Small images display smaller versions of large images, popular in online catalogs for alternative product views. Large images can be unwieldy, so small gallery views are preferred. HTML5 extends beyond basic markup to include APIs for complex applications, including native audio and video elements and built-in offline storage mechanisms, allowing full-blown applications to run without a network connection.

Lesson 20: Getting Started with JavaScript Programming

What You'll Learn in this Lesson

  • How and why to organize scripts using functions
  • What objects are and how JavaScript uses them
  • How JavaScript can respond to events
  • How and when to use conditional statements and loops
  • How browsers execute scripts in the proper order
  • Basic syntax rules for avoiding JavaScript errors
  • What JSON is and how it can be used

A statement is a single action code section in a JavaScript program, serving as the fundamental units of the program. A statement in JavaScript can be broken across multiple lines or include multiple statements in a single line. A semicolon marks the end, but can be omitted if starting a new line. A function is a unit of JavaScript statements, used in a function call. A function can be defined by an expression enclosed in parentheses and can also return a value to a waiting variable. JavaScript variables are containers that can store numbers, text strings, or other values. JavaScript supports objects, which store data like variables. These objects provide predetermined actions, like document.write, and store properties like name, address, and telephone number for specific purposes.

JavaScript is case sensitive, meaning lowercase and capital letters cannot be used interchangeably. Keywords are always lowercase, built-in objects are capitalized, DOM object names are usually lowercase, and uppercase letters start all words except the first one. Variable names in JavaScript can be uppercase or lowercase, with the same name being used each time. Variable names should not be reserved words, including if and for. JavaScript comments provide brief documentation within a script, useful for understanding it after a break. They can be started with two slashes or in the middle of a line, and support C-style comments. To write valid JavaScript, use comments liberally, use semicolons at the end of each statement, use external JavaScript files, avoid being browser-specific, and keep JavaScript optional. This strategy, known as progressive enhancement, ensures easier reading and debugging, faster page loading, and reusable modular scripts.

JavaScript Best Practices

  • Use comments liberally
  • Use a semicolon at the end of each statement and use only one statement per line
  • Use external JavaScript files whenever possible
  • Avoid being browser specific
  • Keep JavaScript optional

JavaScript allows client-side scripts to dynamically manipulate web browser and documents, using a treelike structure within the DOM, without requiring server requests. The window object represents a browser window, with distinct instances of the browser. Multiple windows open simultaneously, each with its own document. JavaScript references the currently in focus window. The history object is a property of the window object that contains information about previous and next visited locations, including methods to access them. The location object is a child of the window object, storing the URL loaded in the browser window, as demonstrated in a JavaScript statement.

The modern DOM organizes DOM objects under the document object for every element of a page, enhancing understanding of the organization of important DOM objects. In the DOM, each element is a node, with relationships between nodes represented by lines. Nodes can have parents and children, using the same terminology as JavaScript. Text nodes work differently, with paragraph text being a child of the <p> object. The DOM organization of objects is known as siblings, which refer to objects with the same parent, indicating they are at the same level in the DOM object tree. The DOM structure allows for control over elements in web pages, such as paragraphs and images. Prior to W3C DOM and CSS2, only layers were repositioned. Layers can be created using container elements. A script can dynamically modify a heading's content within a web page by assigning its object to the variable heading1, which contains its text and the firstChild property.


Common DOCUMENT Object Properties

document.URL Specifies the document's URL, which cannot be altered by the user or the code.
document.title Refers to the title of the current page, defined by the HTML <title> tag. You can change the value of this property.
document.referrer Returns the URL of the page the user was viewing before the current page—usually the page with a link to the current page. As with document.URL, you cannot change the value of document.referrer. Note that document.referrer will be blank if a user has directly accessed a given URL directly.
document.lastModified Indicates the date the document was last modified. This date is sent from the server along with the page.
document.cookie Enables you to read or set a cookie used within the document.
document.images Returns a collection of images used in the document.

Common DOCUMENT Object Methods

document.getElementById() Returns the element with the specified ID.
document.getElementsByClassName() Returns an array-like object of elements with the specified class.
document.getElementsByTagName() Returns an array-like object of elements with the specified tag name.
document.querySelector() Returns the first element that matches the specified CSS selector.
document.querySelectorAll() Returns a list of elements that match the specified CSS selector.
document.createElement() Creates an element with the specified tag name.
document.createTextNode() Creates a text node with the specified text.
document.write() Writes content directly to the document (rarely recommended).
document.open() Opens a new document stream.
document.close() Closes the document stream.
document.adoptNode() Adopts a node from another document.
document.importNode() Imports a node from another document.

Basic Node Properties

nodeName The name of the node (not the ID). For nodes based on HTML tags, such as <p> or <body>, the name is the tag name: p or body. For the document node, the name is a special code: #document. Similarly, text nodes have the name #text. This is a read-only value.
nodeType An integer describing the node’s type, such as 1 for normal HTML tags, 3 for text nodes, and 9 for the document node. This is a read-only value.
nodeValue The actual text contained within a text node. This property returns null for other types of nodes.
innerHTML The HTML content of any node. You can assign a value including HTML tags to this property and change the DOM child objects for a node dynamically.

Variables are named containers for data storage, with unique names chosen by the user. They can include alphabetic letters, digits, underscores, spaces, and underscores. The first character must be a letter or underscore. Variable names are case sensitive and have no official limit. Longer names may require a different naming convention. JavaScript uses the var keyword to declare variables, which can be omitted, but they are still declared when assigning a value. the equal sign (=) is used to assign a value to a variable. JavaScript offers two types of shorthand for this syntax: the += operator, which creates a shorter version of the preceding example, and the −= operator, which subtracts a number from a variable. The increment and decrement operators, ++ and −−, are also included. An expression is a JavaScript expression that combines variables and values to evaluate to a single value, with operators like + and / used to combine these values. JavaScript uses operator precedence rules to calculate values using multiple operators. The highest precedence operator is evaluated first, ensuring the desired result. However, this can sometimes result in incorrect results, which can be controlled using parentheses. JavaScript supports basic data types like Number, Boolean, String, and Null. These types allow JavaScript to store integers, logical values, strings, and undefined variables, ensuring efficient data handling.

JavaScript is a programming language that facilitates data conversions between different data types. JavaScript stores strings as String objects, similar to other variables. Two ways to create a new String object are using built-in functions and object-oriented syntax. Common techniques for working with strings include assigning strings to variables. Assigning a value to a string is similar to assigning an initial value to any other variable, and can be done after the string has been created. The substring method returns a string with a portion of the original string between two index values, specified in parentheses, like the string test string. An array is a numbered group of data items, containing strings, numbers, objects, or other types. To create a numeric array, declare it and assign values using an index. The sort method doesn't work with numeric arrays alphabetically. A function can be specified in the sort method's parameters to correctly sort a numeric array. JavaScript supports functions to organize scripts, making them easier to read. Functions are groups of JavaScript statements that can be used as a single unit. JavaScript creates an array using the new keyword, utilizing built-in functionality and properties like location.href and length property to store values within the object.

Conditional statements

A conditional statement allows a script to perform different actions based on whether a specified condition evaluates to true or false . In JavaScript, the primary tool for implementing a conditional statement is the if statement. A variety of conditional operator exists, including:

== Is equal to
!= Is not equal to
< Is less than
> Is greater than
>= Is greater than or equal to
<= Is less than or equal to

Quick decisions can be made using JavaScript's shorthand conditional expression, a syntax similar to C, in addition to the if statement. variable= (condition) ? (value if true) : (value if false);

An example of using multiple conditional statements

if (hour_of_day < 10) {
document.write("Good morning.");
} else if ((hour_of_day >= 14) && (hour_of_day <= 17)) {
document.write("Good afternoon.");
} else if (hour_of_day >= 17) {
document.write("Good evening.");
} else {
document.write("Good day.");
}

Although this construction is logical, it can get messy if each if statement has its own block of code with several statements. Instead, JavaScript's switch statement lets you combine multiple variable or expression tests into one block

switch(button) {
case "next":
    window.location="next.html";
    break;
case "previous":
    window.location="previous.html";
    break;
case "home":
    window.location="home.html";
    break;
case "back":
    window.location="menu.html";
    break;
default:
    window.alert("Wrong button.");
}

Looping

The for keyword is a crucial tool for creating loops, which use a counter or index to track the number of times the loop has executed and stops when the counter reaches a certain number. The for loop consists of three parameters: the Initial expression, which assigns an initial value to a variable; the Condition, which remains true for the loop to continue; and the Increment expression, which executes with each iteration of the loop to increment the counter.

One possible use of looping could be to count up or down by printing numbers in a sequence. For example:

for (i=1;i<10;i++) {
    document.write("This is line " + i + "<br>");
}

While loops in JavaScript execute statements as long as a condition is true, executing even if the condition starts as false. The while statement is a block of statements within braces, similar to a for loop, with the condition enclosed in parentheses.

while (total < 10) {

n++;

total += values[n];

}

Simple for and while loops are suitable for simple tasks, but complex loops require careful considerations. Avoid infinite loops and use break and continue statements for more control. for and while loops provide control over loops, but can cause problems if not handled carefully.

JavaScript offers a ...in loop, designed to perform operations on an object's properties, like displaying the user's browser's properties in the built-in navigator object.

Event handler

A script that detects and responds to an event is called an event handler. Powerful JavaScript features include event handlers. A useful event handler often requires only one statement, making them one of the easiest features to learn and use. The DOM has event handlers for detecting mouse actions, including mouse pointer movement and button clicks.

Click handler

The onclick event handler is a basic event used to detect when the mouse button is clicked while positioned over the appropriate object. Here is an example of a "click" handler:

<a href="http://www.google.com/" onclick="alert('You are about to leave this site.');">
    Go Away
</a>
Click here

In the example, the click event handler triggers a JavaScript alert before loading a linked page, enabling conditional links or disclaimers before redirecting the user to the linked page.


Keystrokes handler

JavaScript detects keyboard actions using keypress, keydown, and keyup event handlers. Keypress events reference the key being pressed and send its value back to the script, while keydown events do not. To know which key the user pressed, use the event object, which stores the ASCII character code for the pressed key. If you prefer to deal with actual characters, use the fromCharCode String method to convert a numeric ASCII code to its corresponding string character.

You can make your page listen for keystrokes by using the addEventListener method in JavaScript.

.addEventListener("keypress", displayKey);

Load events

The load event is a common event used when the current page, including its images, finishes loading from the server. It is defined using an event handler in the tag, like this:

window.addEventListener("load", function() {
alert("Loading complete");
});
An anonymous function is used for short-term or multiple locations.

Window objects

Users may have multiple window objects in their browser session if they have multiple tabs open or a web page has opened a pop-up window. All open window objects have the document, history, and location objects for scripting.

Each window object has the following useful properties:

window.closed Indicates whether the window has been closed. This makes sense only when you’re working with multiple windows because the current window contains the script and cannot be closed without ending the script.
window.name Specifies the name for a window opened by a script.
window.outerHeight and window.outerWidth Specify the height and width, respectively, of the outside of a browser window.
window.screen Provides a child object that stores information about the screen the window is in—its resolution, color depth, and so on.
window.self Acts as a synonym for the current window object.

The DOM allows you to manipulate and resize windows created by your scripts using various methods for a window object. JavaScript's window.setTimeout method allows for "sleeping" or "nothing at all" for a specific time. It specifies a time delay and a command to execute after it passes, allowing the browser to continue.

The window object has three methods for displaying messages and interacting with the user: alert, confirm, and prompt. These methods display dialog boxes, prompt input, and use a variable for response.
